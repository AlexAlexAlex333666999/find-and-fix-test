Tutorial: Inspect your Java code to find and fix errors

This tutorial describes how to find and fix errors in your Java application with a help of Inspect Code. This feature allows you to get a list of errors of your program, export the list, quickly fix the issues, and many more.

You can also watch a tutorial video about the feature at the end of the page.

What is an error?

In our case, an error is any difference between an expected and actual behavior of your code. The expected behavior may come from requirements for the program, industry standards, or your own perception.

Code example

As an example, letâ€™s take a small program that asks users for a name and country and then greets them.

We intentionally put several errors into the code.

import java.util.Scanner;
class Main {
    public static void main(String[] args) {
        Scanner myObj = new Scanner(System.in);

        System.out.println("What's your name?")
        String name = myObj.nextLine();

        System.out.println("Where areyou from?")
        country = myObj.nextLine();

        System.out.println("Hello, " + name + " from " + country + !);
    }
}

Open the code in IntelliJ IDEA

If you open the code in IntelliJ IDEA, you will see clues telling you about the errors: the code editor will contain a few red lines and one green line, the country variable will be written in red, the scrollbar will display red stripes.

For the small program like ours, these hints would probably be enough. However, if you are working with a piece of code that takes hundreds of lines, you might want to inspect the whole piece with one click and see a list of errors. Maybe you would like to export the list as well. That is when the Inspect Code feature comes into play.


Run an inspection

To start an inspection of your code:

  1. Right-click anywhere in the code editor.
  2. Select Analyze | Inspect Code. The Specify Inspection Scope dialog will be displayed.

Specify the inspection scope

In this dialog:

  1. Specify the inspection scope by choosing between:
    • Whole project: inspection of a whole project.
    • File ‘file_path’: inspection of the opened file only (chosen by default).
    • Custom scope: a scope that you can either choose from a list or create a new one by clicking Shift+Enter and operating with the Scopes window.

    You can also tick Inspect injected code.

  2. Specify the inspection profile. You can either:
    • Choose it from the list.
    • Click Configure and, in the Inspection window, set up the checks that you would like to perform. The window allows you not only include and exclude tests but also assign it to a scope and set up both severity and highlighting style.


  3. Click Analyze. Messages in the status bar mean the inspection is running. If there are no errors, the Code inspection did not find anything to report message will be displayed at the right bottom of the screen. In our case, the Problems window with a list of errors will be opened.

View and export the inspection results

If you inspect our code example, the Problems list will contain one typo and six errors: three for both Java annotator and Syntax error groups.

On the left from the error list, you can see a toolbar.

There, you are able to expand the list of errors by either clicking each group one by one or clicking Expand All ( or Ctrl+NumPad+).

You can also group the list by severity () and/or directory ().

Plus, you are able to export the list by clicking Export () and choosing a format among HTML, XML, and Sarif (JSON).


Ignore the errors

You can also remove the errors from the report. There are various ways to do that. In particular, you can:

  • Exclude an error by clicking its name and Delete on a keyboard. Alternatively, you can right-click the error and select Exclude. The error will be removed from the list but will reappear on the next report.
  • Suppress an error by clicking its name and selecting Suppress for class, Suppress for method, or Suppress all inspections for class. The @SuppressWarnings(“Check group”) line will be added to the code, the error will be deleted from the list and will not appear on the next report until you remove the suppress warning from your code.

Fix the errors

To fix the errors:

  1. Click an error name on the Problems list. Preview of the error will be displayed on the right from the list.
  2. If you would like to highlight the line in the code editor, right-click the error and choose Jump to Source or click F4. Alternatively, you can tick Navigate with Single Click () in the toolbar. This way, the code editor will jump to the relevant line every time you click the error.

  3. Click Apply a quickfix ( or Alt+Enter) to see if there are any suggestions available.
  4. If the quickfix did not help, highlight the code line and follow the tips suggested in the code editor and scrollbar stripes.
  5. If there were no automatically proposed solutions, analyze the hints. There is a high chance they will give you an idea on how to resolve a problem.

Letâ€™s fix the issues by doing the following:

  • Close the lines 6 and 9 with semicolons.
  • Write are you instead of areyou on the line 9.
  • Declare a variable country as a string on the line 10.
  • Close the exclamation mark (!) with quotation marks (“”) on the line 12.

As a result of the edits, our code looks like this:

import java.util.Scanner;

class Main {
    public static void main(String[] args) {
        Scanner myObj = new Scanner(System.in);

        System.out.println("What's your name?");
        String name = myObj.nextLine();

        System.out.println("Where are you from?");
        String country = myObj.nextLine();

        System.out.println("Hello, " + name + " from " + country + "!");
    }
}

Rerun the inspection and run the program

After the edits, some errors in the list are marked as No longer valid. Still, we cannot be sure that our program runs smoothly. To check the errors before starting the program, rerun the inspection. To do that, click Rerun Inspection in the toolbar ( or Ctrl+F5).

For our edited code, the Code inspection did not find anything to report message will be displayed at the right bottom of a screen. It means we are good to go. To run the program, you can either:

  • Click Run ‘Main’ (
  • Select Run | Run ‘Main’ from the main menu.
  • Select Run | Run from the main menu.
  • Click Shift+F10.

In our case, there is no difference between these options. You can find more information on how to run applications in the Running section.

Since we fixed the previous issues, the program will run smoothly with the output like this:

What's your name?

Frodo

Where are you from?

the Shire

Hello, Frodo from the Shire!

Did Inspect Code find every error?

Unfortunately, we cannot guarantee that the code inspection finds all errors. Clean code from the inspection standpoint does not mean the program is error-free. For example, even our simple program passed the last code inspection, but still has some flaws. For instance, it does not capitalize neither name nor country, it does not ask a user to enter their details again in case of zero input, and so on.

Remember that some of the errors are not issues from a technical standpoint but they contradict a logical human perception of your application. Such errors are tough to find during the development. However, IntelliJ IDEA provides several solutions that will help you with this challenging task.
