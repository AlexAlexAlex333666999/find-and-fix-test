Tutorial: Inspect your Java code to find and fix errors

This tutorial describes how to find and fix errors in your Java code with a help of Inspect Code. This feature allows you to get a list of errors in your program, export the list, quickly fix the issues, and many more.

You can also watch a tutorial video about the feature at the end of the page.

What is an error?

In our case, an error is any difference between an expected and actual behavior of your code. The expected behavior may come from requirements to the program, industry standards, or your own perception.

Code example

As an example, let’s take a small program that asks users for a name and country and then greets them.

We intentionally put several errors into the code.

import java.util.Scanner;
class Main {
    public static void main(String[] args) {
        Scanner myObj = new Scanner(System.in);

        System.out.println("What's your name?")
        String name = myObj.nextLine();

        System.out.println("Where are you from?")
        country = myObj.nextLine();

        System.out.println("Hello, " + name + " from " + country + !);
    }
}

Open the code in IntelliJ IDEA

If you open the code in IntelliJ IDEA, you will see clues telling you about the errors: the code editor will contain a few red lines, the country variable will be marked as red, the scrollbar will display red stripes.

For the small program like ours, these hints would probably be enough. However, if you are working with a piece of code that takes hundreds of lines, you might want to inspect the whole piece with one click and see a list of errors. Maybe you would like to export the list as well. That’s when the Inspect Code feature comes into the play.


Specify the inspection scope

In this dialog:

  1. Specify the integration scope by choosing between:
    • Whole project: inspection of a whole project.
    • File ‘file path’: inspection of the opened file only (chosen by default).
    • Custom scope: a scope that you can either choose from a list or create a new one by clicking Shift+Enter and operating with the Scopes window.

    You can also tick Inspect injected code.

  2. Specify the inspection profile. You can either:
    • Choose it from the list.
    • Click Configure and set up the checks that you would like to perform in the Inspection window. The window allows you not only include and exclude tests but also assign it to a scope and set up both severity and highlighting style.
  3. If needed, click the question icon. It opens the Specify Inspection Scope dialog page in our help center.
  4. Click Analyze. Messages in the status bar mean the inspection is running. If there are no errors, the Code inspection did not find anything to report message will be displayed at the right bottom of the screen. In our case, the Problems window with a list of errors will be opened.

View and export the inspection results

If you inspect our code example, you will see six errors: three for both Java annotator and Syntax error groups.

On the left from the error list, you can see a context menu.

There, you are able to expand the list of errors by either clicking each group one by one or clicking Expand All (screen6 or Ctrl+NumPad+).

You can also group the list by severity (screen7) and/or directory (screen8).

Plus, you are able to export the list by clicking Export (screen9) and choosing a format among HTML, XML, and Sarif (JSON).


Ignore the errors

You are also able to remove errors from the report. There are various ways to do that. In particular, you can:

  • Exclude an error by clicking its name and Delete on a keyboard. Alternatively, you can right-click the error and select Exclude. The error will be removed from the list but will reappear on the next report.
  • Suppress an error by clicking its name and selecting Suppress for class, Suppress for method, or Suppress all inspections for class. The @SuppressWarnings(“Check group”) line will be added to the code, the error will be deleted from the list and will not appear on the next report until you remove the suppress warning from your code.

Fix the errors

To fix the errors:

  1. Click an error name on the Problems list. Preview of the error will be displayed on the right from the list. If you would like to highlight the line in the code editor, right-click on the error and choose Jump to Source or click F4. Alternatively, you can tick Navigate with Single Click in the context menu. This way, the code editor will jump to the relevant line every time you click the error, like it is shown on the video below.
  2. Click Apply a quickfix ( or Alt+Enter) to see if there are any suggestions available.
  3. If the quickfix didn’t help, highlight the line in the code editor and follow the tips suggested in the editor and scrollbar stripes, like it is shown on the video below.

If there were no automatically proposed solutions, analyze all hints. There’s a high chance they will give you an idea on how to resolve a problem.

As you could see on the video above, we resolved three issues thanks to the IntelliJ IDEA clues. In particular, we:

  • Declared a variable country as a string on the line 11.
  • Closed lines 7 and 10 with semicolons.

As a result of the edits, our code looks like this:

import java.util.Scanner;

class Main {
    public static void main(String[] args) {
        Scanner myObj = new Scanner(System.in);

        System.out.println("What's your name?");
        String name = myObj.nextLine();

        System.out.println("Where are you from?");
        String country = myObj.nextLine();

        System.out.println("Hello, " + name + " from " + country + !);
    }
}

Rerun the inspection and run the program

After the edits, some errors in the list will be marked as No longer valid. Still, we cannot be sure that our program runs smoothly. To check the errors before running the program, rerun the inspections. There are three ways to do that. You can:

  • Do it with the same settings by clicking Rerun Inspection in the context menu (screen12 or Ctrl+F5).
  • Change the inspection profile by clicking Settings in the context menu (screen13) and setting up a scope in the Inspection window.
  • Rerun a particular test group (syntax errors, for example) by right-clicking anywhere inside the group in the Problems list, selecting Run inspections on…, specifying a scope and clicking Analyze. The video below shows this process.

If you reinspect our code with the same settings, you will see there is one error left. Specifically, the line 13 has the issue.

System.out.println("Hello, " + name + " from " + country + !);

As you can see, we have not closed the exclamation mark (!) with quotation marks (“”). The edited line will look like this:

System.out.println("Hello, " + name + " from " + country + "!");

If you change the line 13 to the one above and rerun the inspection, you will see the Code inspection did not find anything to report message at the right bottom of a screen. As a result, the program will run smoothly with the output like this:

What's your name?

Frodo

Where are you from?

the Shire

Hello, Frodo from the Shire!

Did Inspect Code find every error?

Unfortunately, we can’t guarantee the code inspection finds all errors. Clean code from the inspection standpoint doesn’t mean the program is error-free. For example, even our simple program passed the last code inspection, but still has some flaws. For instance, it doesn’t capitalize neither name nor country, it doesn’t ask a user to enter their details again in case of zero input, and so on.

Remember that some of the errors aren’t issues from a technical standpoint but they contradict a logical human perception of your applications. Such errors are tough to find during the development. However, IntelliJ IDEA provides several solutions that will help you with this challenging task.
