Tutorial: Inspect your Java code to find and fix errors

This tutorial describes how to find and fix errors in your Java code with a help of Inspect Code. This feature allows you to get a list of errors in your program, export the list, quickly fix the issues, and many more.

You can also watch a tutorial video about the feature at the end of the page.

What is an error?

In our case, an error is any difference between an expected and actual behavior of your code. The expected behavior may come from requirements to the program, industry standards, or your own perception.

Code example

As an example, let’s take a small program that asks users for a name and country and then greets them.

We intentionally put several errors into the code.


import java.util.Scanner;
class Main {
    public static void main(String[] args) {
        Scanner myObj = new Scanner(System.in);

        System.out.println("What's your name?")
        String name = myObj.nextLine();

        System.out.println("Where are you from?")
        country = myObj.nextLine();

        System.out.println("Hello, " + name + " from " + country + !);
    }
}

Open the code in IntelliJ IDEA

If you open the code in IntelliJ IDEA, you will see clues telling you about the errors: the code editor will contain a few red lines, the country variable will be marked as red, the scrollbar will display red stripes.

For the small program like ours, these hints would probably be enough. However, if you are working with a piece of code that takes hundreds of lines, you might want to inspect the whole piece with one click and see a list of errors. Maybe you would like to export the list as well. That’s when the Inspect Code feature comes into the play.


Specify the inspection scope

In this dialog:

  1. Specify the integration scope by choosing between:
    • Whole project: inspection of a whole project.
    • File ‘file path’: inspection of the opened file only (chosen by default).
    • Custom scope: a scope that you can either choose from a list or create a new one by clicking Shift+Enter and operating with the Scopes window.

    You can also tick Inspect injected code.

  2. Specify the inspection profile. You can either:
    • Choose it from the list.
    • Click Configure and set up the checks that you would like to perform in the Inspection window. The window allows you not only include and exclude tests but also assign it to a scope and set up both severity and highlighting style.
  3. If needed, click the question icon. It opens the Specify Inspection Scope dialog page in our help center.
  4. Click Analyze. Messages in the status bar mean the inspection is running. If there are no errors, the Code inspection did not find anything to report message will be displayed at the right bottom of the screen. In our case, the Problems window with a list of errors will be opened.

View and export the inspection results

If you inspect our code example, you will see six errors: three for both Java annotator and Syntax error groups.

On the left from the error list, you can see a context menu.

There, you are able to expand the list of errors by either clicking each group one by one or clicking Expand All (screen6 or Ctrl+NumPad+).

You can also group the list by severity (screen7) and/or directory (screen8).

Plus, you are able to export the list by clicking Export (screen9) and choosing a format among HTML, XML, and Sarif (JSON).
